CHESS REMAKE — SESSION CONTEXT
================================
Last updated: 2026-04-10

PROJECT GOAL
------------
Online multiplayer chess in Python. Two milestones per CLAUDE.md:
  1. Standard chess with all legal rules (DONE — current state)
  2. Later: mod the game for an assignment (DO NOT implement until asked)

The rules engine in shared/ must stay overridable so mods can change
representation, rules, space, time, uncertainty, or number of players
without touching client/server code.

TECH STACK
----------
- Client:  Python + Pygame
- Server:  FastAPI + uvicorn + WebSockets
- Engine:  python-chess wrapped behind shared/engine.py
- Tests:   pytest
- Lint:    ruff (line-length=99, select=E,F,I,N,W)
- Types:   mypy (disallow_untyped_defs=true; pygame.* ignored)
- Python:  3.11+ (venv uses python3)

RUNNING THE GAME
----------------
  source venv/bin/activate
  uvicorn server.main:app --reload         # terminal 1
  python -m client.main                    # terminal 2
  python -m client.main                    # terminal 3 (auto-pairs with #2)

Server listens on ws://localhost:8000/ws. Window is 860x640 (640 board +
220 sidebar). Two clients auto-pair via the matchmaking queue.

KEY ARCHITECTURE NOTES
----------------------
shared/engine.py
  - ChessEngine wraps python-chess.Board
  - GameResult(winner: Optional[bool], reason: str)
    winner True=white, False=black, None=draw
  - game_result() returns GameResult for checkmate / stalemate /
    insufficient material / fivefold repetition / 75-move rule.
    It does NOT auto-return for claimable draws (50-move, threefold) —
    those require can_claim_draw() + an explicit claim.
  - is_promotion_move(from_sq, to_sq) + promotion_move(..., promotion)
  - Static square helpers (square, square_file, square_rank, etc.)

shared/models.py
  - Pydantic v2 discriminated unions on "type" field
  - Client→Server: JoinQueue, MakeMove(uci), Resign, OfferDraw,
    AcceptDraw, DeclineDraw, ClaimDraw
  - Server→Client: QueueWaiting, GameStart(color, fen, game_id),
    GameState(fen, last_move, is_check, turn, legal_moves, move_stack,
    can_claim_draw), GameOver(winner, reason, fen), MoveError,
    DrawOffered, DrawDeclined, ServerError
  - parse_client_message(data) routes dict → correct model

server/main.py
  - Globals: waiting_queue (deque), active_games (dict),
    ws_to_game (id(ws) → game_id)
  - GameSession holds ChessEngine + two PlayerConnection objects +
    draw_offer_from (color that offered)
  - try_pair() pairs two queued sockets, sends GameStart + broadcasts state
  - handle_move / resign / draw_offer / accept_draw / decline_draw /
    claim_draw
  - Disconnect handling: forfeit to opponent, cleanup session
  - /health returns {"status": "ok"}
  - Uses Literal["white", "black"] for winner field (mypy fix)

client/main.py
  - ChessClient: engine (shared), renderer, WS connection, selection/drag
    state, promotion state, draw offer state
  - flipped = True when playing black
  - Async recv_messages with 0.001s poll timeout inside pygame loop
  - Sidebar buttons: resign, offer_draw, claim_draw, accept_draw,
    decline_draw, new_game
  - Calls self.renderer.load_assets() in __init__ after BoardRenderer()

client/renderer.py
  - BoardRenderer.load_assets() loads 12 PNGs via
    pygame.image.load(path).convert_alpha() + smoothscale to
    int(square_size * 0.85). On any missing file or load failure,
    returns early so _images_loaded stays False and fallback is used.
  - _draw_piece() uses loaded image if available, else
    _draw_fallback_glyph() (colored circle with letter K/Q/R/B/N/P).
  - draw_board / draw_pieces / draw_dragging_piece / draw_highlights /
    draw_promotion_dialog all go through _draw_piece.
  - square_to_pixel / pixel_to_square handle board flipping for black.

KNOWN GOTCHAS / PRIOR FIXES
---------------------------
- `python` not found on macOS — use `python3` for venv creation.
- Unicode chess glyphs do not render on this Mac's system fonts
  (segoeuisymbol/arial unicode ms/dejavu sans lacked them) — showed
  as empty rectangles. Fixed by switching to PNG images with letter
  fallback.
- mypy: GameOver winner needs Literal["white", "black"], not plain
  Optional[str]. See server/main.py _game_over_msg.
- ruff: keep lines ≤ 99 chars. Common culprits are long docstrings
  and ternary assignments — split into if/else.
- pyproject.toml: mypy override only for pygame.*. Do not re-add a
  chess.* override (unused → warning).
